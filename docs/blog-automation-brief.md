## 執筆ルール

- 文字数: 1800〜2800字程度
- 構成: H2見出し3〜5個、必要に応じてH3。導入文で悩みに共感し結論を先出しする
- タイトルには「熊本」と「パーソナル」または「ジム」のどちらかを自然に含める
- 本文中に主要キーワードを自然に散りばめる（不自然な連呼・詰め込みはしない）
- ブランドファクト（上記）に基づく正確な情報のみ記載し、誇張・断定的な医療効果表現（「必ず痩せる」等）は使わない
- 記事末尾に無料体験への内部リンクを設置: `<a href="https://lp.zennuwellnessdesign.jp/#reserve">無料体験の予約はこちら</a>`
- 敬体（です・ます調）、一人称は「ZenNu」または「私たち」
- bodyはリッチエディタ用の簡易HTML（`<h2>` `<h3>` `<p>` `<ul><li>` `<a>` のみ使用）

## microCMSフィールド

| フィールドID | 内容 |
|---|---|
| title | 記事タイトル |
| slug | 半角英数・ハイフンのみ。トピックを要約した英語スラッグ |
| category | 「トレーニング」「食事指導」「店舗情報」「初心者向け」のいずれか |
| excerpt | 100字程度の要約（一覧カード・meta description用） |
| body | 本文HTML |
| noindex | false固定 |
| eyecatch | 指定しない（コード側でデフォルト画像にフォールバックする） |

## 下書き作成手順（技術メモ）

1. `~/.zennu-lp-secrets/microcms-write-key.txt` からAPIキーを読む
2. `GET https://zennuwellness.microcms.io/api/v1/blogs?limit=100&fields=title,slug` で既存記事一覧を取得し、重複トピックを避ける
3. コンテンツID用に半角英数字のスラッグ（例: `kumamoto-personal-gym-price`）を生成
4. `PUT https://zennuwellness.microcms.io/api/v1/blogs/{生成したスラッグ}?status=draft`
   - ヘッダー: `X-MICROCMS-API-KEY: <キー>` `Content-Type: application/json`
   - ボディ: title/slug/category/excerpt/body/noindexのJSON
5. レスポンスが200番台であることを確認する。失敗した場合はエラー内容をそのまま報告する（黙って諦めない）
